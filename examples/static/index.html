Launch web apps on Nebari — static or Python, from a form, API, or agent.

Point the Nebari Apps Pack at some HTML and it becomes a real app on your cluster: a Deployment, a Service, and routing with TLS and Keycloak SSO — all from one declarative App resource. This very page is one of those apps.

Four ways to launch

Every producer writes the same App custom resource, so launch semantics are identical whether a person clicks a form or an agent types a sentence.

ui
The apps-ui

A dashboard with analytics, an app detail view with logs and events, and a launch form with direct zip / .html upload.

api
The REST API

Programmatic CRUD plus logs, events, and status — the single authority the UI and MCP are thin clients over.

agent
A coding agent

Connect the in-cluster MCP server and launch with natural language: “create a public site called release-notes,” then “launch it.”

gitops
kubectl / GitOps

Apply the App CR straight to the cluster. The API writes CRs directly, so GitOps is optional, never required.

The pieces

One contract, a small set of components. Each producer converges on the App CR; the operators turn it into a running, routed app.

App CRD

apps.nebari.dev/v1alpha1

The declarative contract every producer writes.

apps-operator

Go · controller-runtime

Reconciles App → Deployment, Service, NebariApp.

apps-api

Python · FastAPI

CRUD, observability, and zip / .html upload.

apps-ui

React · TS · Vite

Dashboard, launch form, app detail with logs.

apps-mcp

Python · FastMCP

Agent tools at /mcp; Keycloak device-flow auth.

new-nebari-app

Claude Code skill

Scaffold an app + nebari-app.yaml, then “launch it.”

How this page got here

No build step, no server to babysit, no auth code. This page is a static App the pack is serving right now at team-site.apps.nebari.test.

1
Declare an App

One App resource carries the files — inline, or from a git repo or pvc.

2
apps-operator reconciles

It materializes the files and creates a Deployment, a Service, and a NebariApp.

3
Routed automatically

nebari-operator turns the NebariApp into an HTTPRoute, TLS, Keycloak SSO, and a landing-page tile.

team-site.yaml
# This page — an inline static App. Delete it and
# the Deployment, Service, route, and TLS all cascade.
apiVersion: apps.nebari.dev/v1alpha1
kind: App
metadata:
  name: team-site
spec:
  displayName: "Nebari Apps Pack"
  source:
    type: inline
    inline:
      files:
        index.html: "<!doctype html>…"
  access:
    public: true
    subdomain: team-site
